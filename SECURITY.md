# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in MaCursor, please report it responsibly. **Do not open a public issue or pull request.**

- **Email:** [contact@writronic.com](mailto:contact@writronic.com)
- **GitHub:** [Report a vulnerability](https://github.com/writronic/MaCursor/security/advisories/new) via private vulnerability reporting

Please include:
- A description of the vulnerability
- Steps to reproduce
- Affected version(s)

## Supported Versions

Only the latest release is supported with security updates.

| Version | Supported |
|---------|-----------|
| Latest  | ✅        |
| Older   | ❌        |

## Disclosure Process

1. We will acknowledge your report within **7 days**.
2. We will investigate and work on a fix privately.
3. A patched release will be published, and the vulnerability disclosed in the release notes.

We ask that you allow us a reasonable timeframe to address the issue before any public disclosure.

## Contact

- **GitHub Issues:** [github.com/writronic/MaCursor/issues](https://github.com/writronic/MaCursor/issues) *(for non-security bugs only)*
